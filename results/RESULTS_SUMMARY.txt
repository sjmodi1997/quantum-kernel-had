METHODOLOGY
================================================================================
Step 1: Classical Feature Extraction
  - Compute Transfer Entropy (directional causality) between banks
  - Risk Virality Score = PageRank on TE network (super-spreader identification)
  - Hyperedge degree = co-exposure metrics
  - Create feature vector [TE_mean, RVS_max, HDEG_mean] per 252-day window

Step 2: Quantum Feature Mapping
  - Normalize features to [0, π] range
  - Encode into 6-qubit parameterized circuit
  - Create superposition of quantum states

Step 3: Quantum Kernel Computation
  - For each pair of windows (i, j):
    * Prepare state |ψ(x_i)⟩
    * Apply inverse circuit |ψ(x_j)⟩†
    * Measure overlap probability |⟨ψ(x_i)|ψ(x_j)⟩|²
  - Build symmetric 245×245 kernel matrix

Step 4: One-Class SVM Training
  - Train SVM with precomputed quantum kernel
  - Decision boundary separates normal from anomalous windows
  - Nu=0.05: identify ~5% extreme anomalies

Step 5: Evaluation
  - Compare quantum predictions against ground-truth crisis labels
  - Compute F1, Precision, Recall, AUC-ROC
  - Compare with classical baseline

QUANTUM ADVANTAGE ANALYSIS
================================================================================
Why Quantum Kernels Capture Contagion Better:

1. Higher Dimensionality (Classical: 3D → Quantum: 64D)
   - More room to separate crisis/normal patterns
   - Classical linear boundaries insufficient

2. Non-Linear Feature Interactions
   - CNOT entanglement creates cross-feature terms
   - Captures complex contagion dependencies

3. Quantum Interference
   - Kernel computed via amplitude interference
   - Sensitive to phase differences (not just magnitudes)
   - Phases encode temporal causality (TE → lag information)

4. Implicit Feature Mapping
   - No need to explicitly construct 64D space
   - SVM operates in quantum feature space via kernel
   - Computational advantage: kernel in O(n²), not O(2^6 × n²)

LIMITATIONS & CAVEATS
================================================================================
1. Simulator Noise:
   - Classical simulation tracks all 2^6=64 amplitudes
   - No hardware noise/decoherence, but limited by classical simulation precision

2. Feature Encoding:
   - Only 3 classical features → 6 qubits
   - Information bottleneck: quantum circuit can't exceed classical input
   - Potential for more features if feature count increased

3. Circuit Depth:
   - 3 layers chosen for balance (expressivity vs runtime)
   - Deeper circuits could improve but increase computation time

4. Hyperparameter Tuning:
   - SVM Nu parameter: fixed at 0.05
   - Could optimize via cross-validation (future work)

5. Generalization:
   - Results on Indian market 2015-2025
   - Transferability to other markets/time periods: unknown
